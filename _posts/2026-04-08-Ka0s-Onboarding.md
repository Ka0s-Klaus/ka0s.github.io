Ka0s es un framework “code-first”.

Eso significa que el entorno de desarrollo no es un detalle: es parte del sistema.

En `core/docs/ka0s_onboarding/01_getting_started.md` se fijan requisitos claros:

- Python 3.10+ para automatización.
- Node 18+ para tooling.
- Docker para pruebas locales.
- Git + SSH (y GPG para maintainers).
- `gh` (GitHub CLI) como herramienta diaria.

## El objetivo del onboarding

No es imponer herramientas por capricho.

Es reducir variabilidad para que:

- las ejecuciones sean reproducibles,
- los errores sean diagnosticables,
- y el equipo opere con el mismo “contrato” técnico.
